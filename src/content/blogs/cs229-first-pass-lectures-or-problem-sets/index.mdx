“先把课全部看完，再开始做作业”听起来很顺，但对 CS229 往往无效。课程视频会制造强烈的熟悉感：老师的每一步都能跟上，公式也似乎看懂了；一关掉视频，自己却不知道应该从哪个假设开始。

反过来，完全不看课就直接做 Problem Set，也容易把时间耗在符号、背景和题目意图上。第一遍真正要解决的，不是课程与作业谁排在前面，而是**两者之间的反馈周期有多长**。

![看课、推导、作业与回看组成的三遍学习循环](/images/blog/cs229-learning-system/three-pass-loop.webp)

## 结论：先看一小段，再尽快做一道题

推荐的最小循环是：

```text
浏览讲义与题目 15 分钟
→ 看一个主题的课程 60–90 分钟
→ 闭卷复述 15 分钟
→ 做对应题目 30–60 分钟
→ 定点回看
→ 写一个最小实现或检查
```

关键是把“看课到做题”的间隔压到同一天或第二天。不要等一整章结束，也不要等所有课程播放完。

## 为什么不能先把课看完

看课主要训练识别，作业要求的是回忆、选择和组合。两者的认知任务不同：

- 看见老师使用对数似然时，你可以理解这一步；
- 面对新题时，你要先判断应该写似然还是损失；
- 推导卡住时，你还要定位缺的是概率规则、矩阵微分还是模型假设。

如果反馈延迟两周，届时你只知道“不会做”，却很难知道当初是哪一层没有学稳。短循环让错误仍然带着上下文出现，修正成本更低。

## 看课前：先给大脑几个挂钩

不需要完整预习。只做三件事：

1. 浏览本节讲义的小标题；
2. 看对应 Problem Set 在问什么，但不看解答；
3. 写下两个问题，例如“为什么使用这个分布”“这个优化有没有闭式解”。

这 15 分钟让课程从连续的信息流变成带任务的搜索。你会更容易注意到老师何时回答了自己的问题。

## 看课时：只记决策，不抄黑板

笔记应该围绕模型设计决策：

| 要记的内容 | 示例                               |
| ---------- | ---------------------------------- |
| 问题设定   | 输入、输出、监督信号是什么         |
| 建模假设   | 噪声分布、条件独立、共享协方差     |
| 目标函数   | 最大化似然还是最小化经验风险       |
| 求解方法   | 闭式解、梯度法、Newton's Method    |
| 失败模式   | 小样本、共线性、过拟合、数值不稳定 |

长推导只标起点、关键变换和终点。完整重建留给课后；否则你会把大量注意力用在跟上书写速度，而不是理解为什么要这样建模。

## 课后 15 分钟：闭卷画出一条链

关掉所有材料，写出：

```text
数据 → 假设 → 概率模型 → 目标函数 → 梯度/解法 → 预测
```

空白本身就是诊断结果。如果你只能写出最终公式，说明模型的来路没有建立；如果能写出概率模型但推不到目标函数，应该回看似然部分，而不是重播整节课。

## 做题时：使用 30 分钟规则

一道题独立尝试 25 到 40 分钟。卡住后，不要立刻看完整答案，而是先写清楚：

- 已知什么；
- 要证明或计算什么；
- 当前做到哪一步；
- 缺少哪一条定义或技巧。

然后只查足以解除当前阻塞的内容。查到后合上资料继续。这样保留了题目的检索与组合训练，也避免在一个技巧上无效消耗几小时。

## 三种常见卡点，对应三种回看方式

### 概念卡点

表现为不知道模型为什么这样假设。回看课程中的动机和几何直觉，必要时找一个二维例子，不要先钻进代数。

### 推导卡点

表现为知道路线，但等号走不下去。回到公式卡片或推导 Note，检查条件概率、链式法则和 shape。

### 实现卡点

表现为公式正确但代码不收敛。先做数值梯度、损失曲线和极端输入检查，不要假设问题一定来自数学。

## 一周 8 小时的第一遍模板

| 时段     | 工作               | 输出                   |
| -------- | ------------------ | ---------------------- |
| 2 小时   | 预习并看课         | 半页主题地图           |
| 1.5 小时 | 精读讲义并闭卷推导 | 一份推导 Note          |
| 3 小时   | Problem Set        | 解题过程和错误分类     |
| 1 小时   | 最小实现           | 可运行脚本与一张检查图 |
| 0.5 小时 | 复盘               | 学习日志与下周第一步   |

如果只有 4 小时，把主题减半，不要把作业和复盘删掉。完整走完一个小主题，比半懂不懂地看完三个主题更有价值。

## 什么时候可以进入下一主题

第一遍不追求熟练，但应满足四个最低标准：

1. 能用自己的话说出模型解决的问题和主要假设；
2. 能在提示很少的情况下重建主要推导路线；
3. 能独立完成至少一道代表性问题；
4. 能运行一个最小实现，并解释一张结果图。

不需要等到所有细节都记住。卡点应被写入 Notes，错误应进入日志；只要主干已经建立，就可以继续，并在后续模型的比较中加深理解。

## 最后：让作业决定你回看什么

第一遍最危险的不是慢，而是没有反馈。课程负责提供地图，讲义负责补逻辑，作业负责揭露假理解，代码负责把结论变成可检查的行为。

所以答案不是“先做作业”或“先看课”，而是：**先看足以开始做题的内容，然后尽快让题目告诉你下一段课应该怎么看。**

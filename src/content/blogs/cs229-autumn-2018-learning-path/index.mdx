CS229 的难点不只是公式多，而是材料分散：视频、讲义、课程 notes、Problem Sets、补充数学资料和代码实现各自都很完整，却没有告诉你每天应该先打开哪一个。

如果只是把所有链接收藏起来，很容易得到一个“资料很全、进度为零”的文件夹。更有效的做法，是把它们组织成一个学习系统：**Page 放地图，Blog 放阶段性结论，Notes 放会持续修订的知识原子，代码仓库保存可运行的证据。**

![CS229 学习系统：课程、推导、作业与代码仓库](/images/blog/cs229-learning-system/hero.webp)

这篇文章给出一套面向 CS229 Autumn 2018 的完整组织方式。目标不是追求“看完”，而是让每一个主题最终都留下四类产物：直觉、推导、实现和复盘。

## 一、先决定内容放在哪里

这套内容不应该全部挤在一篇长文里。不同材料的更新频率和阅读方式不同，适合放在不同栏目。

| 位置     | 放什么                                                 | 为什么                                                   |
| -------- | ------------------------------------------------------ | -------------------------------------------------------- |
| Page     | CS229 课程总入口、仓库地址、进度、目录和所有子内容链接 | 它是一张长期维护的地图，不依赖发布日期                   |
| Blog     | 完整、可独立阅读、有明确结论的阶段总结                 | 适合从首页、分类页和标签页被发现                         |
| Notes    | 学习日志、公式卡片、单个概念推导和容易反复修改的材料   | 适合按目录生长，也允许保留半成品思考                     |
| Git 仓库 | 作业代码、实验、环境、测试和勘误                       | 代码应当可以运行、比较和追溯，而不是粘在文章里失去上下文 |

建议的 Page 标题是 **CS229 Autumn 2018 学习中心**，slug 使用 `cs229-autumn-2018`。Page 顶部放仓库链接，下面依次放当前进度、学习路线、Blog 文章和 Notes 索引。它不是又一篇文章，而是所有 CS229 内容的单一入口。

课程资料与作业代码使用 [maxim5/cs229-2018-autumn](https://github.com/maxim5/cs229-2018-autumn)。这个链接同时放在系列入口，读者不必先进入某一篇文章才能找到原始仓库。

### Blog 规划

1. **CS229 Autumn 2018 资料目录与学习路线**：也就是本文，解决“材料有哪些、按什么顺序学”。
2. **CS229 第一遍学习路线：先做作业还是先看课**：记录真实执行后的节奏调整，回答输入与输出怎样配比。
3. **CS229 Problem Set 1 总结**：围绕线性分类、概率解释、推导和代码中的典型错误做一次闭环复盘。

### Notes 规划

- `cs229/learning-log`：CS229 学习日志。
- `cs229/formula-sheet`：CS229 公式卡片。
- `cs229/logistic-regression`：Logistic Regression 推导笔记。
- `cs229/gda-and-logistic-regression`：GDA 和 Logistic Regression 的关系。
- `cs229/kernel-trick`：Kernel trick 笔记。

Notes 的目录名保持英文和稳定，标题可以是中文。这样 URL 不会随着措辞调整而变化，Page 和 Blog 中的链接也更可靠。

## 二、资料目录：哪些是主线，哪些只是补充

学习 CS229 时，资料越多不一定越好。先给每一种材料分配角色，避免同一个主题重复看四遍。

### 1. 课程视频：负责建立第一层直觉

视频的任务不是让你记住每一步推导，而是回答三个问题：

- 这个模型试图解决什么问题？
- 为什么要做这些假设？
- 损失函数、概率模型和优化目标之间怎样连起来？

第一遍看课允许有公式没跟上。只要能在课后用自己的话说出问题设定、核心假设和最终目标，就可以进入讲义，而不是立刻倒回去重看整节课。

### 2. Lecture Notes：负责补齐逻辑链

讲义是学习主线，不只是视频字幕。阅读时应主动标出：

- 定义和符号；
- 推导从哪一个假设开始；
- 哪一步使用了概率、线性代数或微积分；
- 最终得到的训练目标和预测规则；
- 与前一个模型相比改变了什么。

如果一段推导读不动，先把卡点写成具体问题，例如“为什么 Bernoulli 对数似然会变成交叉熵”，不要只记“这里不懂”。具体问题可以直接进入对应的 Note。

### 3. Problem Sets：负责暴露假理解

作业不是学完后的考试，而是学习材料本身。看懂一页讲义只能证明你能跟随作者；关掉讲义重新推一次，才说明符号和逻辑已经成为自己的工具。

对每一道题，建议保留四层答案：

1. 题目要求证明或实现什么；
2. 使用哪个定义或假设；
3. 中间推导或代码；
4. 结果如何检查。

最后一层最容易被忽略。维度是否一致、概率是否归一、梯度方向是否合理、极端输入下是否符合直觉，都是不依赖标准答案的检查手段。

### 4. 代码仓库：负责保存可验证的学习成果

推荐让仓库与课程主题对齐，而不是与文件类型对齐：

```text
cs229-autumn-2018/
├── README.md
├── environment.yml
├── pset1/
│   ├── linearclass/
│   ├── poisson/
│   └── src/
├── pset2/
├── pset3/
├── experiments/
│   ├── logistic-boundary/
│   ├── gda-vs-logistic/
│   └── kernel-features/
└── tests/
```

[原始课程仓库](https://github.com/maxim5/cs229-2018-autumn)负责保存课程材料与作业代码；个人实验可以 fork 后按上面的结构补充。README 只记录环境、运行方式、完成状态和博客入口；长篇解释留给 Blog，随时修订的推导留给 Notes。仓库中的每个实验最好有一个最小命令可以复现结果，而不是只保存 notebook 最后一次运行的输出。

## 三、正确的顺序不是“看课或做题”二选一

更可靠的节奏是一个短循环：

```text
预习问题 → 看课建模 → 讲义补推导 → 闭卷复述 → 做题暴露缺口 → 回看 → 写最小实现
```

![看课、推导、作业与回看组成的三遍学习循环](/images/blog/cs229-learning-system/three-pass-loop.webp)

### 第一遍：建立地图，不追求推导零缺口

先浏览主题、作业题目和讲义小标题，再看课程。这里的完成标准是能写出一张半页纸的概念图：输入是什么，输出是什么，模型作了什么假设，训练在优化什么。

时间建议：一节课的 1 到 1.5 倍。暂停次数过多通常意味着把第一遍误当成了精读。

### 第二遍：重建推导，不照抄讲义

合上讲义，从问题设定开始写。卡住后只打开到足以继续的位置，再合上。对 Logistic Regression，至少应独立走通这条链：

$$
h_\theta(x)=\sigma(\theta^T x)
\quad\Rightarrow\quad
p(y\mid x;\theta)
\quad\Rightarrow\quad
\ell(\theta)
\quad\Rightarrow\quad
\nabla_\theta \ell(\theta).
$$

写完后再问：为什么这是判别式模型？如果从 $p(x\mid y)$ 出发，边界会怎样出现？这个问题自然连接到 GDA Note。

### 第三遍：让作业和代码决定回看哪里

先独立做 25 到 40 分钟。只有当你能准确说出卡点，才回到视频或讲义。回看的单位应是一条定义或一小段推导，而不是整节课。

代码实现则用于验证：

- 数值梯度是否与解析梯度一致；
- 损失是否按预期下降；
- 决策边界是否随正则化改变；
- 特征缩放是否影响优化速度；
- 生成式与判别式模型在小样本下表现如何。

## 四、12 周学习路线

这是一条偏扎实、可以边工作边执行的路线。若每周时间不足，优先保持循环完整，宁可把一周拉长，也不要只看课不做输出。

| 周次 | 主线主题                                       | 本周必须留下的输出                      |
| ---- | ---------------------------------------------- | --------------------------------------- |
| 0    | 线性代数、概率、微积分热身；环境与仓库         | 环境可复现；写出向量/矩阵求导检查清单   |
| 1    | Linear Regression、Normal Equation、LMS        | 从高斯噪声推到最小二乘；实现 batch/SGD  |
| 2    | Logistic Regression、Newton's Method           | 独立推似然、梯度和 Hessian；画决策边界  |
| 3    | GDA、Naive Bayes、Event Models                 | 比较生成式与判别式假设；做小样本实验    |
| 4    | SVM、Margin、Kernels                           | 写清 primal/dual 角色；实现两个核的对比 |
| 5    | Model Selection、Regularization、Bias/Variance | 用学习曲线诊断欠拟合和过拟合            |
| 6    | Neural Networks、Backpropagation               | 手推两层网络梯度并做数值梯度检查        |
| 7    | EM、Mixture of Gaussians                       | 解释 Jensen's inequality 在 EM 中的位置 |
| 8    | PCA、ICA、Factor Analysis                      | 用几何语言解释降维；比较重构误差        |
| 9    | Reinforcement Learning、MDP                    | 写出 Bellman 方程并实现 value iteration |
| 10   | 作业收尾与综合实验                             | 清理代码、补测试、统一符号和实验记录    |
| 11   | 第二遍复盘                                     | 完成公式卡片、错题分类和主题关系图      |

每周结束时，用同一组问题验收：

- 我能否不用术语堆砌，解释这个模型为什么存在？
- 我能否从假设推到目标函数，而不是只背最终公式？
- 我能否实现一个最小版本，并用测试或图形检查结果？
- 我能否指出它与前后模型的共同结构和关键差异？

四个问题中有两个答不上来，这个主题就不应标成“完成”。

## 五、Problem Set 1 应该怎样做成一次完整复盘

PSet 1 的价值不只是完成若干题，而是第一次把监督学习的四层结构接起来：

```text
数据与假设 → 概率模型 → 优化目标 → 数值实现
```

做题时维护一张错误表：

| 错误类型 | 典型表现                               | 修正动作                       |
| -------- | -------------------------------------- | ------------------------------ |
| 符号错误 | 把样本索引、特征索引和参数维度混在一起 | 每次推导先写 shape             |
| 概率错误 | 忘记条件、归一化或独立性假设           | 把联合分布完整展开一次         |
| 微分错误 | 链式法则漏项、行列向量方向不一致       | 用标量形式和数值梯度双检       |
| 优化错误 | 符号写对但更新方向相反                 | 先判断在做 ascent 还是 descent |
| 工程错误 | notebook 可跑但无法从头复现            | 增加固定入口、依赖和最小测试   |

完成 PSet 1 后，Blog 总结不应逐题抄答案，而应回答：哪些知识缺口重复出现、哪些检查方法最有效、下一轮学习节奏应该怎样调整。涉及课程作业答案的部分保持解释性，不直接发布可复制的完整解答。

## 六、五份 Notes 怎样互相连接

### 学习日志：记录决策，而不是流水账

每次只记录日期、主题、产出、卡点和下一步。最有价值的是“为什么调整路线”，例如：Newton's Method 的代数能跟上，但 Hessian 的几何直觉不足，因此下一次先画二次近似再继续作业。

### 公式卡片：公式必须带使用条件

不要只写 $\nabla_\theta \ell(\theta)$。每张卡片还要包含变量 shape、概率假设、优化方向、数值注意点和与相邻公式的关系。公式卡片的目标是快速重建推导，而不是制造一张更密集的墙纸。

### Logistic Regression：从概率到优化

这份 Note 专注一条完整推导链，并补上 sigmoid 导数、对数似然、交叉熵、梯度、Hessian 和 Newton 更新。每一个等号旁边写“为什么”，比多做几遍机械微分更有效。

### GDA 与 Logistic Regression：共享边界，不共享建模方向

GDA 建模 $p(x\mid y)$ 和 $p(y)$，再通过 Bayes 得到 $p(y\mid x)$；Logistic Regression 直接建模 $p(y\mid x)$。在共享协方差的高斯假设下，GDA 会导出线性的 log-odds，但这不意味着两个模型相同。Note 应重点比较假设、样本效率、模型偏差和边界形式。

### Kernel trick：计算内积，而不是显式制造无限特征

Kernel Note 从二次特征映射的具体例子开始，再进入 $K(x,z)=\phi(x)^T\phi(z)$。每遇到一个 kernel，都问它隐含了什么相似性，以及算法是否只通过样本内积依赖特征。

## 七、一个现实可执行的每周模板

如果每周有 8 到 10 小时，可以这样分配：

| 环节        |     时间 | 产物                         |
| ----------- | -------: | ---------------------------- |
| 预习与看课  | 2.5 小时 | 半页概念图和问题清单         |
| 精读与推导  |   2 小时 | 一份可复查的推导 Note        |
| Problem Set |   3 小时 | 解题过程、卡点和检查记录     |
| 实现与实验  | 1.5 小时 | 可运行代码、图和最小测试     |
| 周复盘      |   1 小时 | 学习日志、公式卡片和下周入口 |

如果只有 4 小时，不要删除作业和复盘，而是缩小主题范围。只看两节课会制造进度感；完成一个主题的闭环会留下可复用的能力。

## 八、最终完成标准

CS229 的“完成”不是播放列表全部变灰。更可信的完成标准是：

1. Page 可以让陌生读者在五分钟内理解你的路线和当前进度；
2. 三篇 Blog 分别回答资料组织、首轮节奏和 PSet 1 复盘；
3. 五份 Notes 可以独立阅读，也能互相链接；
4. 仓库从干净环境可以运行主要作业和实验；
5. 随机抽一个模型，你能解释假设、推导目标、实现核心算法并说出失败模式。

当这些产物都存在时，课程视频只是输入，真正留下来的是一套能迁移到其他机器学习课程和论文的学习方法。

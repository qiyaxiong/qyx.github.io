## 一条贯穿整份作业的主线

无论题目最终要求推导还是编程，都可以还原成五层：

```text
问题与数据
→ 建模假设
→ 目标函数
→ 优化方法
→ 数值与结果检查
```

最常见的问题，是直接从第三层开始背公式。这样一旦损失函数稍微换一种写法，或者标签从 $\{0,1\}$ 变成 $\{-1,1\}$，整条推导就会失去方向。

## 1. 先把 shape 写出来

在写任何梯度前，先固定记号：

$$
X\in\mathbb{R}^{m\times n},\qquad
\theta\in\mathbb{R}^{n},\qquad
y\in\mathbb{R}^{m}.
$$

然后逐项检查：$X\theta$ 是 $m$ 维预测，$X^T(X\theta-y)$ 是 $n$ 维梯度。shape 检查不能证明公式正确，却能在最早阶段排除大量转置和广播错误。

代码里也应坚持同一约定，不要让一部分函数使用 `(m,)`，另一部分隐式依赖 `(m, 1)`。很多“算法不收敛”实际只是数组形状触发了静默广播。

## 2. Logistic Regression 的核心不是 sigmoid

sigmoid 只是把线性分数映射为概率：

$$
h_\theta(x)=\sigma(\theta^Tx)=\frac{1}{1+e^{-\theta^Tx}}.
$$

更重要的是从 Bernoulli 条件模型写出似然：

$$
p(y\mid x;\theta)=h_\theta(x)^y(1-h_\theta(x))^{1-y}.
$$

对样本取对数并求和，才得到训练目标。这样推导的好处是，每个符号都有概率含义；梯度不再是凭空出现的一条公式。

实现时应使用数值稳定版本的 sigmoid 和 log-loss。对于很大的正负 logits，直接计算指数或先算概率再取对数都可能溢出。稳定性是算法的一部分，不是额外的工程细节。

## 3. Gradient 与 Newton 更新要先确认方向

如果写的是 log-likelihood，通常执行 ascent；如果写的是 negative log-likelihood，则执行 descent。两份代码可能公式只差一个负号，却都正确。真正危险的是推导使用一个约定、更新代码使用另一个约定。

Newton's Method 进一步使用 Hessian 的曲率信息：

$$
\theta \leftarrow \theta-H^{-1}\nabla J(\theta).
$$

实现时不应显式求逆，而应求解线性方程。除了数值更稳定，也更清楚地表达我们需要的是满足 $H\Delta=\nabla J$ 的步长 $\Delta$。

## 4. 生成式与判别式的比较要回到假设

Logistic Regression 直接建模 $p(y\mid x)$。GDA 则建模 $p(x\mid y)$ 与 $p(y)$，再通过 Bayes 得到后验。

在类别条件高斯且共享协方差时，GDA 的 log-odds 对 $x$ 是线性的，因此也得到线性决策边界。但两者并没有变成同一个模型：GDA 作了更强的分布假设，可能在小样本且假设近似正确时更有效；Logistic Regression 的假设更少，数据充足时通常更稳健。

作业中的比较题不应只写“一个生成式，一个判别式”，还要说明差异如何影响参数估计、样本效率和模型偏差。

## 5. 一套不依赖标准答案的检查清单

### 推导检查

- 每一项维度是否一致；
- 概率是否在 $[0,1]$；
- 极端输入下结果是否符合直觉；
- 梯度为零时是否对应合理的驻点；
- 改变标签编码后符号是否仍一致。

### 代码检查

- 用有限差分比较解析梯度；
- 从零参数开始，第一步更新方向是否降低损失；
- 打乱样本顺序后结果是否基本一致；
- 同时画 loss curve 与 decision boundary；
- 增加一个很小的合成数据集，让结果可以人工判断。

### 复现检查

- 从干净环境能否安装依赖；
- 是否有单一运行入口；
- 随机种子和数据路径是否明确；
- notebook 是否可以 restart-and-run-all；
- 输出图是否能由命令重新生成。

## 本次最值得记录的五类错误

| 错误                       | 根因                       | 以后怎样提前发现            |
| -------------------------- | -------------------------- | --------------------------- |
| 梯度正负号相反             | 混用最大化似然与最小化损失 | 更新前写清 ascent/descent   |
| 转置位置错误               | 没有固定样本与特征轴       | 推导开头标 shape            |
| Newton 法不稳定            | 显式求逆或 Hessian 病态    | 使用线性求解并检查条件数    |
| 损失出现 `nan`             | sigmoid/log 计算不稳定     | 用稳定函数并测试极端 logits |
| 图看起来正确但结果不可复现 | 状态隐藏在 notebook        | 提供脚本入口与最小测试      |

## PSet 1 完成后应该留下什么

一份真正完成的作业，不只有提交文件。还应留下：

1. 一页错误分类；
2. Logistic Regression 的闭卷推导；
3. GDA 与 Logistic Regression 的比较 Note；
4. 一个数值梯度测试；
5. 一条从干净环境复现实验的命令。

这些产物会直接复用到后续的 SVM、神经网络和概率模型。PSet 1 的意义，正是建立从假设到实现、再从实现回查推导的工作方式。
